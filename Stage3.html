Lesson 1 : Intro to Abstraction

What are Python Functions and what do they do?

Functions are predefined packages of coding that the programmer can use within thier code. This helps reduce the amount of code the programmer has to write and simplifies the readability of the code. The details of the functions are hidden which allows the programmer to focus on the program instead of coding them. This is called abstraction.

Python Functions

Some examples of Python functions are webbrowser, time, and os. These functions stored in modules that actually exist in the Python library and have to be imported or called up to the Python file to be used. These modules can have specific sub attributes that can be used once the module has been imported. Such as the time function has a sleep attribute which allows suspending of program for a set time that the programmer can define. The time function also has ctime attribute which retrieved current time from the PC.

Lesson 2: OOP - Object-Oriented Programming

Using Classes in Programming

A class is a blueprint that contains data and methods which can be used to create an instance or object. These class modules have to be imported to the program file just like functions had to be imported. The programmer will assign a name to the instance which then would be referred as an object of the class. In the Python library there is a class called Turtle and is used for drawing shapes. We could name this instance brad then define the pencolor, fillcolor, and speed. Then name another instance jeff and define the same attributes with different values. Now then there are two objects of Turtle, one named brad & the other named jeff. These attributes (pencolor, fillcolor, and speed) from each object are called instance variables. These are similiar because they belong to same class but unique to each object because of different name and values assigned. Then the programmer could use each object within the program to draw objects or make designs. This is a basic model of object oriented programming. All classes contain a method called __init__ which is referred to as the constructor. This method contains the variables which will be the instance variables for each object created. There is a default variable of self which is the object's name.

Making Classes

The Python library has a large selection of classes predefined to use but a programmer can build them too. The class name should always start with a capital letter per Google Python Style Guide. Class variables should be in all capital letters per the Google Python Style Guide. Class variables are a set of constant values which can be used by all objects of the class. There are default class variables such as __doc__, __name__, and __module__ which are on all classes. When building classes the programmer can inherit methods from another class. This is called inheritance. This is very similiar to a child inheriting from a parent. The child class could inherit the parent __init__ method and add variables for it's own __init__ method for a combination of both. However the child class could completely define a method with same name as method found in the parent class which is called method overriding. The child class would use that method instead of the inherited parent method.

What is OOP- Object Oriented Programming?

Pure definition of object-oriented programming is a programming paradigm (pattern) based on the concept of objects which are data structures that contain data known as attributes and code in form of procedures known as methods. In object-oriented programming computer programs are designed by making them out of objects that interact with one another.

Benefits of OOP for Large Programming Projects

There are several benefits of OOP because of the predined classes or previous built classes. First is allow the programmer to focus on overall program and it eliminates them from having to program every detail. This same concept of prepackaged code helps in other ways too. Such as less code to write will speed up production and development of a working program. When there is less code to write that will reduce the possibilities for errors too. This prepackaged/modular code will make it easier to maintain changes to the program later on. Instead of making mass changes throughout a program just updating a class makes this much simplier. Since code is being reused it reduced the development time which reduces the cost to develop software.
